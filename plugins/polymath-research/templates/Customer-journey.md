# Customer journey: {{title}}

For persona: **{{persona}}** (link to the Persona doc).

A customer journey is **one persona's path through one job-to-be-done**.
Multiple personas → multiple journeys. Don't merge.

## The job-to-be-done

In one sentence using the JTBD form: "When <situation>, I want to <motivation>,
so I can <outcome>."

## Stages

5–7 stages from before they encounter your product through the outcome they
care about. Stages are user-perspective, not product features.

| Stage | What they're doing | What they're thinking/feeling | Touchpoints | Pain |
| ----- | ------------------ | ----------------------------- | ----------- | ---- |
| 1. Trigger | … | … | (mostly outside your product) | … |
| 2. Search | … | … | Google, peer asks | … |
| 3. Evaluate | … | … | docs, sales call | … |
| 4. Onboard | … | … | signup, first run | … |
| 5. First success | … | … | the feature | … |
| 6. Habit | … | … | repeat use | … |
| 7. Tell others | … | … | shares, referrals | … |

## Peak pain moments

The 1–2 moments in the journey where the persona is most likely to drop off.
These are the leverage points.

## Where we have evidence vs. assumptions

Mark every stage:

- **✓ Evidence** — quote from an interview supports this.
- **? Assumption** — we're guessing; needs research before acting.

A journey full of assumptions is a journey on a whiteboard, not a journey.

## Improvements (sized by impact, not by ease)

Top 3 changes that would move the peak pain moments, with the metric each
would shift.
